- 这是一个门户网站的后台管理
- 其中门户管理是一级菜单。后台功能他们都需要在ruoyi-portal模块下。前端页面都应该在portal目录中。
- 要注意做好controller还有mvc mapping的目录分级。
- java文件修改后，及时移除无效的import
- 原则上不要使用在代码中使用全路径类名，确实有类名冲突时，才使用全路径类名。
- 有些功能需要清洗或初始化数据的，你可以使用skills查看表中数据，根据需要进行清洗或初始化。
- 对于功能上线所需要的初始化数据sql，原则上使用insert or update语义，避免反复执行sql导致数据重复的问题。
- 你应该给vue、java的前后端工程文件目录的同级加上模块名称，这样也方便你后续找模块。
- 接口编制完以后，你需要验证好功能，确保前端调用的接口保持一致，避免前端调用错误的接口。
- 你一定要明确问题以后再进行修改，尤其是涉及到一些字段、变量命名，你认为不匹配，但是一定要仔细分辨。如果是不确定的，就先不该。一定要充分论证，确保修改不会影响到其他功能。
- **前端组件name与菜单route_name一致性规范**：前端Vue组件中 `script setup name` 的值必须与数据库 `sys_menu.route_name` 完全一致（区分大小写），否则 `keep-alive` 缓存无法生效。常见正确命名：`WeComChannel`、`DingtalkChannel`、`FeishuChannel`、`EmailChannel`、`MessageChannel`。
- **数据库字段定义与前端代码必须一致**：SQL建表语句中的字段类型和长度必须与前端实际存储的值匹配。例如：`template_type` 前端存储的是 `notification/reminder/alert` 等字符串（长度>1），就不能定义为 `CHAR(1)`，而应使用 `VARCHAR(20)`。每次编写SQL前必须确认前端代码中该字段的实际值。涉及字段类型变更时，还需要同步修改 `upgrade_*.sql` 和 `portal_init.sql` 中的定义。
- 后台开发的要求是，controller层要放portal-system模块中，按照模块区分，比如system、portal、message等。
- 对于需要移动或重命名Java文件的任务，必须使用 `git mv` 命令，而不是直接 Delete + Write 操作。这样可以保留 Git 历史记录，实现真正的文件移动跟踪。
- 新增或迭代功能模块时，必须同步更新 `sql/portal_init.sql`，将新表的建表语句添加到该文件，确保初始化数据的完整性。
- 对于升级类SQL（如 upgrade_xxx.sql），必须执行该SQL脚本将变更应用到数据库，可使用skills中的mysql-query工具执行。
- 你需要通过数据判定一些事情的时候，优先通过查询数据库来分析。
- **数据库表字段变更规范**：
  - 在已有表结构中做修改时，一个要更新init的初始化sql脚本
  - 二是需要提供更新的SQL脚本（ALTER TABLE ADD COLUMN）
  - **重要**：使用Python脚本执行ALTER TABLE添加字段，避免MySQL不支持的语法
  - 技能使用方式：`python .trae/skills/mysql-query/mysql_query.py --python add_columns.py`
  - Python脚本模板：
    ```python
    import pymysql
    conn = pymysql.connect(host=DB_HOST, port=DB_PORT, user=DB_USER, password=DB_PASSWORD, database=DB_DATABASE, charset=DB_CHARSET)
    cursor = conn.cursor()
    try:
        cursor.execute("ALTER TABLE {table_name} ADD COLUMN {column_name} {definition}")
        conn.commit()
        print('{table_name}.{column_name} 添加成功')
    except Exception as e:
        print(f'错误: {e}')
    cursor.close()
    conn.close()
    ```
